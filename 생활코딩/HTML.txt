HTML 문법





기본 문법

속성 : <__> 이런 것들이 전부 속성에 포함.

1. HTML에서는 시작 속성과 끝 속성을 구분해줘야함.
ex) <h1> </h1> 
이와 같이 슬래시를 통해 끝 속성이라는 것을 나타내줌.
------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

<strong> : 글자를 굵은 글씨로 나타냄.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

<h1> : 제목 표시

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

<a> : anchor의 줄임말. 외부링크로 연결하는 연결고리
------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

href : <a>안에 쓰임. 외부링크의 주소를 나타낸다.
ex) <a href = "https://ko.wikipedia.org/wiki/HTML">HTML</a>
웹사이트에서 HTML이라는 글자를 누르면 위 링크로 이동하게 됨.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

"_blank" : 새로운 탭에서 링크를 열게 함.
ex) (<a href = "https://ko.wikipedia.org/wiki/HTML"  target = "_blank">What is HTML?</a>)
What is Html? 이라는 글씨를 클릭하면 새로운 탭에서 위 링크로 연결 됨.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

title : 웹 페이지나 anchor의 이름을 나타내줌.

ex1)
<title> 
HTML
</title>

↓

웹페이지 이름의 HTML이라고 뜸.


ex2)
(<a title = "HTML 위키피디아 문서">   
What is HTML?
</a>)

↓

What is HTML 위에 커서를 가져다 댔을 때 작게 "HTML 위키피디아 문서" 라고 뜸.
------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

<meta charset - "utf-8"> : 한국어 깨짐 현상이 일어날떄 encoding값을 설정해주는 태그

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------


<ul> : unordered list/순서가 없는 리스트/ ul을 응용해서 공백을 나타낼 수 있음.
ex)
<ul>
  <li>가 나 다 라 마 바 사 아 자 차 카 타 파 하</li>
</ul>

가 나 다 라 마 바 사 아 자 차 카 타 파 하


↓

가 나 다 라 마 바 사 아 자 차 카 타 파 하

가 나 다 라 마 바 사 아 자 차 카 타 파 하

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

<ol> : ordered list/순서가 있는 리스트
ex)
<ol>
<li>가 </li>
<li>나 </li>
<li>다 </li>
</ol>

↓

1. 가
2. 나
3. 다

심화 문법

<html> : head와 body를 감싸는 전체 코드

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

<head> : 페이지의 본문의 쓰이는 내용은 아니지만 본문을 꾸며주거나 기본 값들을 설정해주는 코드의 부분

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

<body> : 본문의 내용이 쓰이는 코드의 부분

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
시멘틱 웹(Semantic web)

문서의 의미 중심 구성
시멘틱 웹은 단순히 데이터를 나열하는 것이 아니라, 그 데이터의 의미를 명확히 하여 문서와 정보의 구조를 구정
이를 통해 컴퓨터와 사람이 모두 쉽게 이해할 수 있는 웹 생성

컴퓨터의 이해와 처리 가능
시멘틱 웹에서는 컴퓨터가 인간의 개입 없이도 정보를 일고, 긔 의미를 파악하며, 새로운 정보를 생성
이넨 데이터 간의 관계를 분석하고, 정보의 의미를 파악함으로써 가능

의미론적 데이터의 추출
시멘틱 웹에서는 정보를 분석하여 그 관계 속에서 의미론적인 자료들을 추출
이를 통해 검색 엔진이나 다른 프로그램들이 더 정확하고 유용한 정보를 제공

<header> : 문서나 섹션의 머리말을 정의하는 태그
사이트나 섹션의 상단 부분에 위치하며, 주제나 내비게이션 링크를 포함

<nav> : 내비게이션 링크를 그룹화하는 태그
사이트의 주요 내비게이션 메뉴를 정의

<main> : 문서의 주요 콘텐트를 감싸는 태그로, 페이지의 핵심 콘텐츠를 정의
블로그 포스트나 기사 등 ㅍ체이지의 주용 내용을 담고 있는 부분

<article> : 독립적으로 배포 가능하며, 자체적으로 의미를 가지는 콘텐츠를 정의
각각의 블로그 포스트, 뉴스기사 등

<section> : 문서 내의 논리적인 구획을 정의하는 태그로, 관련된 콘텐츠를 그룹화
문서의 주요 주제를 구분

<aside> : 문서의 주요 콘텐츠와는 별도로, 관련된 보조 콘텐츠를 정의하는 태그
사이드바에 표시되는 추가 정보나 광고

<footer> : 문서나 섹션의 바닥글을 정의하는 태그
페이지 하단에 위치한 저작권 정보, 사이트 링크, 연락처
